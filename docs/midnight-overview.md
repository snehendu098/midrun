# Midnight Network Overview

## What is Midnight

Privacy-first L1 blockchain built by IOG (the team behind Cardano). Uses zero-knowledge proofs to enable confidential smart contracts while maintaining regulatory compliance. Preprod testnet live now, mainnet targeting late March 2026.

## Dual-State Ledger

Midnight splits state into two domains:

- **Public state** - on-chain, visible to all, consensus-verified
- **Private state** - off-chain, held locally by users, proven correct via ZK proofs

This lets contracts selectively disclose data. Only what you explicitly put in the ledger is public.

## Token Model

| Token | Type | Purpose |
|-------|------|---------|
| **NIGHT** | Unshielded/public | Base token, visible on-chain, transferable |
| **Shielded tokens** | ZK-private | Privacy-preserving transfers, balances hidden via ZK proofs |
| **DUST** | Fee token | Generated by staking NIGHT, used to pay tx fees |

## Compact Language

Smart contracts are written in **Compact**, a TypeScript-like language that compiles to ZK circuits.

### Key Constructs

```compact
export ledger {
  // On-chain public state - visible to everyone
  totalBets: Counter;
  houseBalance: Uint<64>;
}

export circuit bet(amount: Uint<64>): Void {
  // Entry points callable by users
  // Logic here is ZK-proven (enforced correct without revealing inputs)
}

witness playerSecret(): Field {
  // Off-chain private data supplier
  // Data from witness stays confidential
  // NOT enforced by ZK - trust the local execution
}
```

### Confidentiality Rules

- Data **NOT in `ledger`** stays confidential (private by default)
- Computation **NOT in `witness`** is enforced correct via ZK proofs
- `witness` code runs locally, unproven - use for private input injection only

### Data Types

| Type | Description |
|------|-------------|
| `Bytes<N>` | Fixed-length byte arrays |
| `Uint<N>` | Unsigned integers of N bits |
| `Counter` | Incrementable on-chain counter |
| `Field` | ZK field element (native to the proof system) |
| `Opaque<"string">` | Opaque reference to off-chain data |
| `enum` | Sum types / tagged unions |
| `struct` | Product types / records |

### Standard Library

- `transientHash` / `persistentHash` - hash functions (transient = per-tx, persistent = cross-tx)
- `transientCommit` / `persistentCommit` - cryptographic commitments for hiding values on-chain
- Transient variants are cheaper but only valid within a single transaction
- Persistent variants survive across transactions (stored in ledger)
